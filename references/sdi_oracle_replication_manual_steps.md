# Manual Steps to set up Oracle User and Permissions for SDI Replication

These next SQL Commands can either be run in `sqlplus` or `sqldeveloper` as `sys`.

```sql
CREATE USER C##LR_USER IDENTIFIED BY HXEHana1 DEFAULT TABLESPACE SYSTEM;
ALTER USER C##LR_USER QUOTA 100m ON SYSTEM;
CREATE VIEW C##LR_USER.RA_ALL_USERS_VIEW (USER#, NAME, PASSWORD, TYPE#, ASTATUS) AS SELECT USER#, NAME, NULL, TYPE#, ASTATUS FROM SYS.USER$;

GRANT UNLIMITED TABLESPACE TO C##LR_USER;
GRANT CREATE SESSION to C##LR_USER;
GRANT LOGMINING to C##LR_USER;
GRANT EXECUTE_CATALOG_ROLE to C##LR_USER;
GRANT SELECT ANY TRANSACTION to C##LR_USER;
GRANT SELECT_CATALOG_ROLE TO C##LR_USER;
GRANT CREATE DATABASE LINK to C##LR_USER;
GRANT CREATE TABLE to C##LR_USER;
GRANT CREATE VIEW to C##LR_USER;
GRANT SET CONTAINER TO C##LR_USER;
GRANT ALTER ANY TABLE TO C##LR_USER;
GRANT CREATE PROCEDURE TO C##LR_USER;
GRANT SELECT ANY TRANSACTION TO C##LR_USER;
GRANT CREATE TRIGGER TO C##LR_USER;

GRANT SELECT ON SYS.DBA_TABLES TO C##LR_USER;
GRANT SELECT ON SYS.DBA_OBJECTS TO C##LR_USER;
GRANT SELECT ON SYS.V_$DATABASE TO C##LR_USER;
GRANT SELECT ON SYS.MLOG$ TO C##LR_USER;
GRANT SELECT ON SYS.DBA_TABLES;
GRANT SELECT ON SYS.V_$LOG TO C##LR_USER;
GRANT EXECUTE ON SYS.DBMS_LOGMNR TO C##LR_USER;
```
